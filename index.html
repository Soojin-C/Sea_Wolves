PhoneGap


PhoneGap is a mobile application development framework. It is an open source distribution of Cordova and allows users to create a mobile app across any platform using HTML5, JavaScript and CSS.

History

First developed by Nitobi Software Once Nitobi Software was acquired by Adobe, its code was contributed to the Apache Software Foundation, which was later used to create the new project Apache Cordova. Cordova and Phonegap are essentially the same except Phonegap has a different set of plugins and Cordova is always open source.

How to create an App

To start, download the Desktop app and the mobile app

First go to the green plus and click Create new PhoneGap project to create a new app

Then choose the path where the local files will be held, the name of the project and the optional id

Different Parts of the APP

config.xml
  • Global configuration file
  • Controls some of the app’s behavior
  • Root element is widget
    • the app’s reverse domain id
    • Version: version number
    • Author
    • Desription
    More about config
www folder
  • Where to put your code
  • www source directory is reproduced within each platform's subdirectory
platforms
  • What the project works on: android, ios, etc
    • $ cordova platform add android
    • $ cordova build : builds project for all platforms
    • $ cordova build ios
    • To check: $ cordova platform ls
  • do not edit : constant copies over files from the source www folder
plugins
  • (added) plugins are in this directory
  • Syntax: cordova plugin add [PLUGIN ID]
  • Ex: cordova plugin add cordova-plugin-sqlite-2 : SQLite database
    • Usage: sqlitePlugin.openDatabase()
    More Plugins
hooks
  • Special scripts to do special commands
  • Ex:
    • before_platform_add:
      <hook type="before_build" src="scripts/appBeforeBuild.js" />
merges
  • Files in merge will override the file in the same platform in the www file

Why PhoneGap

Pros

  • No need to learn any new languages specific to platform. Uses HTML, JavaScript and CSS
  • Avaliable to use across all platforms
  • In the long run, it will be easier for developers because they don't need to keep up with all the changes on multiple platforms

Cons

  • Native apps are seen as more fluid. Performance on a hybrid app may be slower and the user experience may not be as smooth as a native app
  • Local changes made to the phone, such as a font change, will not be changes for the hybrid app but will for a native app
